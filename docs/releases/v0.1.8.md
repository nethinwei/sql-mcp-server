# v0.1.8

`v0.1.8` 交付 IR Semantics and Provider Conformance：为已承诺的读路径 IR
子集建立可验证语义地基，使"同一受治理查询在不同 Provider 上具有相同逻辑
语义"从架构主张变为有差分证据支撑的声明。它不新增 IR 能力、Provider 或
运行时功能；附带交付 v0.1.7 校准选定的一项契约收紧。

## IR 语义规范

[docs/design/ir-semantics.md](../design/ir-semantics.md) 为读路径算子
（Scan/Select/Project/Aggregate/Sort/Limit/Distinct）与全部 11 个谓词
操作符定死语义：

- bag semantics（重复行保留，Distinct 是唯一显式去重）；
- 谓词三值逻辑（`NULL` 比较为 UNKNOWN 并被过滤、`in`/`not_in` 列表含
  `NULL` 的传播、`NOT` 不复活 UNKNOWN）；
- 聚合边界（`count(*)` vs `count(field)`、聚合忽略 `NULL`、全局聚合空输入
  恒一行、分组聚合空输入零行、`NULL` 分组归一组）；
- documented deviations 表：无法统一的行为（Sort 中 NULL 位置、collation
  大小写、avg 精度、聚合列名、整数溢出）逐项登记并绑定 corpus 回避策略；
  静默差异不允许存在，新 Provider 必须先按表声明偏差再进验收。

## Reference interpreter

新包 `core/relalg/interp` 按规范逐条实现读路径语义，作为 codegen 与
Provider 执行的 oracle。规范中每条语义有对应单元测试（三值逻辑、LIKE、
bag/distinct、排序分页、五个聚合函数的空集与 `NULL` 行为、无效 IR 拒绝），
进入常规 `make test`，无需数据库。

## 跨 Provider differential conformance suite

新包 `internal/conformance`：25 个固定边界用例 + 60 个固定 seed 的有界
随机组合（谓词/排序分页/全局与分组聚合），共 85 个差分用例。同一确定性
fixture、同一 IR，interpreter 结果 vs Provider 经 `codegen.Renderer` 编译
执行的结果，经精确 decimal 归一化后按 bag（无 Sort）或有序（Sort 键唯一
非 NULL）比较。

三库接入 `make test-integration`（`Test{PG,MySQL,OB}Conformance`），本
版本验收运行：PostgreSQL、MySQL、OceanBase 各 85/85 通过，无需新增
documented deviation（corpus 设计已按规范回避已登记偏差）。
[Provider 兼容矩阵](../provider-compatibility.md)中 aggregate 与读路径
语义一致性升级为"真实数据库验证"。

## 契约收紧（v0.1.7 校准选定项）

无谓词聚合拒绝（`AggregateGuard`，`COST_EXCEEDED`）的 hint 从"add a valid
WHERE predicate"改为给出可直接执行的最短修复：`codegen.Compiled` 新增
`PrimaryKey` 元数据，hint 指名主键上的 `is_not_null` 谓词示例。单元测试
锁定新 hint；一轮 pilot sanity 运行（32 任务）确认被拒聚合调用收到新
hint 且修复路径有效（repair rate 88%）。

## 已知限制

- conformance corpus 值域限于整数、decimal、小写 ASCII 字符串与 `NULL`；
  时间/时区/collation/布尔的跨库等价尚未进入 corpus（属规范登记的边界）；
- 覆盖读路径 + 聚合；写算子与事务的精确语义规范留待 L12 逐能力立项补齐；
- interpreter 只承诺错误分类一致，不承诺错误消息文本一致；
- conformance 直接执行编译产物，不经过 cost gate 与 RBAC——治理语义由
  既有 integration/e2e 继续保证，本套件只证明查询语义等价。
