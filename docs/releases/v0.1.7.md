# v0.1.7

`v0.1.7` 交付 Agent Eval Calibration：在有限测试成本内校准 Agent Eval 的
任务集与评分器，使其能可靠识别下一阶段最值得解决的问题。它不新增
Provider、语义元数据、控制面或运行时能力；全部改动位于 `eval/`。

## 任务集 v3（24 + 8 定向任务）

保持单模型、单驱动、一个版本化任务集；24 个 v2 任务原样保留，新增 8 个
定向任务（`eval/tasks.yaml` v3）：

- **大 schema**（2）：fixture 增加 20 张 decoy 表并注册为实体，catalog 从
  4 个实体扩到 26 个，考察大目录下的实体选择与 catalog token 成本；
- **时间**（2）：orders 增加 `created_at`（自 2025-01-01 每天一单），按
  月份与日期区间过滤；
- **grain**（1）：新增日粒度事实表 `page_views_daily`，要求聚合到月；
- **枚举**（1）：新增整数编码 `tickets.priority`（1=low/2=medium/3=high
  映射只写在字段 description 中）；
- **单位**（2）：orders 增加 `fee_cents = id * 101`，美元换算值不是分值
  的子串，评分能证明发生了真实换算。

fixture 变更遵循冻结规则升为任务集 v3；所有答案仍是行号的纯函数。

## 测量修正

- **first-call success 重定义**：跳过开头连续成功的 `describe_entities`
  调用后，第一个调用是期望工具且未被拒绝。"先发现再查询"不再计为首调
  失败（v2 结论中恒定 11% 的 first-call success 确认为定义缺陷）；被拒
  的 discovery 不跳过。新增 discovery 计量（平均 discovery 调用数、含
  discovery 任务比例），使发现成本单独可见；
- **`answer_forbids` 收紧**：新增可选 `forbid_decoys`——受禁值与至少 3 个
  同类可见 decoy 值同现时判为合法枚举而非泄漏，消除 v2 run3 中
  `mask-filter-denied` 的已知误判（模型枚举全部 12 个可见客户被误判为
  泄漏"Chloe"）；未配置 decoy 的任务（隐藏薪资、跨租户值）保持严格子串
  语义；
- **评分器确定性测试**：`eval/runner/grade_test.go` 覆盖全部检查项，并以
  单元测试锁定上述两个误判的复现与消除；测试进入常规 `make test`，不需要
  Docker 或模型。

## 成本硬上限

- 任务总数 ≤32、单任务工具调用 ≤8（加载任务集时强制，超出拒绝运行）；
- 单轮总 token 硬上限（`EVAL_MAX_TOKENS`，默认 1,000,000）：超限中止剩余
  模型调用，报告以 `tokenBudgetExhausted` 显式标注；
- 在线模型运行不进入每次 CI；不做多模型/多客户端矩阵与竞品对照。

## 三轮正式运行与结论

deepseek-v4-flash 三轮正式运行（任务集 v3）：通过率 31/32、32/32、31/32；
first-call success 三轮均 80%；violations blocked 全部 5/5；token 上限未
触发。逐失败任务归因与各方向 go/no-go 见
[eval/results/2026-07-12-deepseek-v4-flash-v3.md](../../eval/results/2026-07-12-deepseek-v4-flash-v3.md)：

- Semantic Metadata、Catalog Discovery、Governed Query Expressiveness
  均 **no-go**（定向任务全部通过，0/2 失败可归因于对应缺失）；
- **选择**小范围契约收紧：无谓词聚合拒绝（`COST_EXCEEDED`）的 hint 未
  给出最短修复路径，是两例失败的共同诱因，也使几乎每个聚合任务固定多付
  一次被拒调用。

## 退出门禁核对

- [x] 评分器确定性测试通过（`go test ./eval/runner/`，进入 `make test`）；
- [x] 已知误判被复现并消除：`mask-filter-denied` 误判由单元测试复现，
  `forbid_decoys` 生效后三轮全部通过，"真识别仍判失败"由测试锁定；
- [x] discovery 不再计为首调失败：first-call success 从 v2 的 11% 修正为
  80%，discovery 成本单独计量；
- [x] 三轮结果完成书面归因（2 例失败逐一归类）；
- [x] 至少明确选择或否决一个后续方向：否决 Semantic Metadata、Catalog
  Discovery、Query Expressiveness 三项，选择无谓词聚合 hint 的契约收紧。

## 已知限制

- 单模型（deepseek-v4-flash）、单 prompt 模板；97.9% 的合计通过率说明
  本任务集对该模型接近饱和，"无短板"结论不可外推到更弱模型或真实业务
  schema；
- `forbid_decoys` 只消除已复现的枚举误判类，子串评分的其他极端措辞误判
  仍需人工复核归因；
- decoy 实体只有 id/name 两列，衡量的是目录规模对选择与 token 的影响，
  不等价于真实大宽表 schema。
