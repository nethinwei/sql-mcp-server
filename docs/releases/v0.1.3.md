# v0.1.3

`v0.1.3` 建立公开采用基线，不改变 MCP 工具或数据库执行语义。重点是让新用户
可以获取、验证、启动并发现已发布服务。

## 发布与供应链

- GitHub Release 提供 Linux、macOS、Windows 的 amd64/arm64 归档；
- GHCR 提供 linux/amd64 与 linux/arm64 OCI image；
- 归档附带 SHA-256 checksum 和 SPDX JSON SBOM；
- checksum、镜像及镜像 SBOM 使用 GitHub OIDC 的 keyless Cosign 签名；
- tag workflow 在发布前运行单元、三库 integration 和 PostgreSQL MCP e2e。

验证命令见 [运行与运维](../operations.md)。

## 5 分钟体验

新增 PostgreSQL 16 Docker Compose 示例，默认只监听 loopback，并使用低权限
reader、字段 ACL、email mask、tenant 行策略和成本闸门。quickstart 同时演示
授权 PK 读取、跨 tenant 空结果、全表读取拒绝及隐藏字段拒绝。

详见 [快速体验](../quickstart.md)。

## MCP 发现

根目录 `server.json` 使用官方 schema，声明 OCI stdio 安装方式，并在允许的
publisher metadata 中记录 streamable HTTP、认证、provider 和安全能力。CI 使用
固定版本 `mcp-publisher` 校验；release workflow 在镜像发布后通过 GitHub OIDC
提交 Registry 并回查版本。

官方 Registry 仍处于 preview。Registry 暂时失败不会撤销已经签名的 GitHub
Release，但该版本在 Registry 可发现前不视为完整完成发布。

魔搭 ModelScope 采用“仅本地可用/分发展示”：根目录 `mcp_config.json`、专用
配置和 `make modelscope-check` 会验证真实 PostgreSQL、stdio、授权读取、mask 与
拒绝路径。Hosted 模式不在本版本范围内。

## Provider 基线

新增 [兼容矩阵](../provider-compatibility.md) 与
[支持版本](../supported-versions.md)。矩阵区分真实数据库验证、核心层验证和未
独立验证；不会把 capability 位或 DBA 外部配置写成应用已提供的硬保证。

OceanBase integration 镜像由 `latest` 固定为
`4.3.5.6-106000012026040916`。

## 兼容性

本版本没有计划内配置或 MCP 契约破坏性变更。现有部署升级前仍应运行
`validate`，并阅读 [安全模型](../security.md) 和 provider 兼容矩阵。

六个平台归档由 GoReleaser 交叉构建并校验 checksum；GA 前未在原生 macOS/Windows
桌面客户端完成独立人工 smoke，该项不作为本版本安全保证。
