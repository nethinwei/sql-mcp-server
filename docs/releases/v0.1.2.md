# v0.1.2

`v0.1.2` 在 `v0.1.1` 结构整理之上，收紧安全与资源边界，并将业务核心迁入
`core/`、数据库 provider 改为可插拔注册。包含若干不兼容变更；升级前请阅读
[迁移说明](#迁移说明)。

## 安全与执行边界

- 成本链拆分为不可关闭的 Safety/Enforcement 与可选 Estimate；
  `cost.enabled: false` 不再关闭写保护、CALL 审核、输入及结果上限。
- MySQL/OceanBase 使用保守 EXPLAIN，在错误、未知计划或全表扫描时 fail closed；
  三种 provider 均装配数据库原生 statement timeout。
- procedure 默认拒绝，须设置 `mcp.trustedProcedure: true` 并命中 reviewed
  `allowTemplates`；新增 procedure 独立结果上限 `maxProcedureRows`。
- cache、feedback、IN/filter/groupBy/aggregate/expand、预算 session 和响应字节
  均增加硬边界；expand 改为分批 IN。
- 热重载改为 drain-before-publish；改变工具发现集合的 reload 要求重启。
- prepared statement 不再锁内执行网络 prepare；singleflight 传播 deadline；
  `rateLimit.rps` 现已实际装配 token-bucket 限流。

### 安全修复

- 修复 aggregate 未脱敏、mask 字段谓词/分组侧信道、数据库错误详情外泄、
  procedure rows 泄漏路径和 commit 失败未 rollback。
- bearer token 改为固定长度摘要恒时比较；角色统一小写规范化；动态 JSON 保留
  大整数精度；审计输入字段脱敏并哈希 transaction token。
- `${file:...}` secret 限制到 `server.secrets.allowedRoots` 并阻止符号链接逃逸；
  扩充 DSN 脱敏。

## 结构变更

- 业务包迁入 `core/`（`config`、`tool`、`cost`、`audit` 等）；外部适配保留在
  `x/`。`.golangci.yml` depguard 强制 `core/` 不得 import `x/`。
- YAML 解码迁至 `x/configyaml`，核心配置包不再直接依赖 `yaml.v3`。
- 数据库 provider 通过 `x/providerregistry` 注册；`x/providers/all` blank
  import 装配内置驱动。新增数据库无需修改 `core/` 或 `x/bootstrap`。
- `core/tool` 按工具拆分实现与测试；`x/bootstrap` 拆分装配逻辑。
- `make fmt` / `make fmt-check` 使用 `internal/fmtcheck`（gofmt + golines，
  文件 ≤800 行、函数 ≤50 行、行宽 ≤120 列）。
- 工作区 `.vscode/settings.json` 为 gopls 配置 `integration,e2e` build tag。

## 不兼容变更

- mask 字段不再允许用于 filter、cursor、group-by、aggregate 或写谓词。
- `maxScannedRows` 被 `maxEstimatedScannedRows` 取代（旧字段暂作 deprecated
  alias）；零值不再能产生无界缓存或 mandatory cost limit。
- 角色在配置与请求入口统一 trim 并转为小写，规范化碰撞会拒绝启动。
- 审计文件格式由 YAML 改为 JSON Lines。
- 直接 import 旧顶层包路径的嵌入方须改为 `core/<pkg>`（例如
  `github.com/nethinwei/sql-mcp-server/core/config`）。

## 迁移说明

从 `v0.1.1` 升级时：

1. 将角色名统一为 trim 后的小写，消除规范化碰撞。
2. 移除对 mask 字段 filter/group-by/aggregate/cursor 的依赖。
3. 将 file secret 移到 `server.secrets.allowedRoots` 下的绝对路径。
4. 为必要 procedure 设置 `mcp.trustedProcedure: true`，审核并加入 CALL
   fingerprint，同时配置数据库用户最小权限。
5. 将预算 `maxScannedRows` 迁移为 `maxEstimatedScannedRows`，并检查新增的响应、
   输入、缓存和事务 timeout 默认值。
6. 若消费审计日志，按 JSON Lines 解析；字段已脱敏，transaction token 为哈希。
7. 若代码直接 import 旧包路径，改为 `core/<pkg>`。

完整配置与信任边界见
[配置参考](../configuration.md)与[安全模型](../security.md)。

## 与 v0.1.1 的关系

MCP 工具列表与对外契约不变；默认安全行为更严格，若干配置字段语义收紧。结构
整理见 [`v0.1.1`](v0.1.1.md)，功能基线见 [`v0.1.0`](v0.1.0.md)。
