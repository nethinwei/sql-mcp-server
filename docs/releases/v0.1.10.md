# v0.1.10

`v0.1.10` 交付 Diagnostic Evaluation and Workload Hardening：将 v4
真实业务负载从最终答案验证升级为可测量自主语义理解、行为选择、治理边界、
过程成本和失败归因的 v5 诊断轨。范围唯一事实源为
[Diagnostic Evaluation 设计](../design/diagnostic-evaluation.md)。

本版本不新增运行时产品能力，不扩展 IR 或 Provider。

## 任务 schema v5 与覆盖矩阵

- 48 个正式任务：21 个冻结 Guided 任务 + 27 个扩展任务；prompt 分布为
  guided 23、natural 22、ambiguous 3。
- `dimensions`、`prompt_level`、`expected_behavior`、同源 `oracle` 与
  confounder 进入版本化 loader；`answer`、`clarify`、`deny`、`qualify`、
  `unsupported` 五种行为均有正式任务。
- 生成器提供 8 个以上 counterfactual oracle；错误值命中后机械归因到
  grain、time、status 或 unit 等能力缺口。
- `make eval-coverage` 从受控维度词表生成
  `fixtures/v4/coverage/matrix.json` 与 `matrix.md`；drift 测试和发布门禁
  锁定任务总数、标签、行为、非 Guided semantic 覆盖及治理负向覆盖。

## 诊断 runner 与报告

- 新增 `make eval-diagnostic`，与回归轨、真实负载轨共享模型端点和确定性
  fixture；单轮最多 48 任务、单任务最多 16 次工具调用、默认 2M token。
- 报告新增行为准确度、治理通过率、自动归因率、oracle 归因率、
  unattributed/manual-review、product-fixable 比例及工具/discovery/token
  成本。
- 修复模型在一次响应中并发给出多个 tool call 时可越过单任务调用上限的
  问题；最终三轮没有任务超过 16 次调用。
- 诊断 profile 增加仅用于治理 suite 的 tenant row-policy 与受限实体别名，
  不改变 v4 负载轨 profile 和预期值。

## 正式评测

deepseek-v4-flash 三轮诊断结果：**41/48、43/48、43/48**；行为准确度
三轮均为 14/19（73.7%），治理通过率均为 11/12（91.7%），三项 violation
任务全部阻止。未归因失败三轮均为 0；完整环境、成本、归因和限制见
[正式结论](../../eval/results/2026-07-12-deepseek-v4-flash-diagnostic-v5.md)。

发布后分流：

- Semantic Metadata、Governed Query Expressiveness、Evidence Envelope：
  no-go；
- Catalog Discovery：继续观察；
- Tool Contract：go，进入设计评估但不直接承诺版本；
- 小集合推断安全：有稳定合成信号，因缺少受监管部署需求继续观察。

回归核对为 32/32，v4 真实负载核对为 21/21；PostgreSQL、MySQL、
OceanBase integration/conformance/workload 差分与 MCP e2e 全绿。

## P1 草案

三个多轮调查场景、medium catalog profile 占位和五个跨角色任务保留为
非默认草案；不计入 v0.1.10 任务数、覆盖矩阵或退出门禁。跨 Provider
诊断任务仍留给 capability model 阶段。

## 迁移

无运行时配置或公开 MCP 工具契约迁移。Eval 自动化可新增：

- `make eval-diagnostic`
- `make eval-coverage`

诊断轨默认使用 `fixtures/v4/profiles/diagnostic.yaml`；`EVAL_CONFIG`
仍可显式覆盖。

## 退出门禁核对

- [x] 48 个正式任务均有结构化 dimensions，覆盖矩阵可重复生成；
- [x] 五种 expected behavior 均有正式任务，non-Guided 与治理负向覆盖达标；
- [x] 至少 8 个任务具有同源 counterfactual oracle；
- [x] 典型 grain/time/status/unit 错误可机械或半机械归因；
- [x] 报告单列 unattributed、manual review、product-fixable 与治理通过率；
- [x] 回归轨 32/32，真实负载轨 21/21，三库差分和 e2e 无回归；
- [x] 公开工具契约兼容，fmt/vet/test/race/lint/docs-check/release checks 通过。

## 已知限制

- 正式数字来自单模型、单 endpoint、单 prompt 模板，不外推到其他模型；
- online model 非确定，公开结果必须使用三轮分布；
- 自动 product-fixable 只包含 oracle/mechanical 归因，manual review 不自动
  归类为 model-only；
- 治理 profile 是合成证据，不替代真实租户或受监管部署验证。
