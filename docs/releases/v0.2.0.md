# v0.2.0 安全与成本边界

本版本以不兼容方式收紧默认安全模型。只有同时满足以下条件，项目才声明数据库
操作处于可控执行包络内：

- mandatory Safety/Enforcement 不可由 `cost.enabled` 关闭；
- 查询受并发、RPS、context、数据库原生 timeout、返回行和返回字节上限约束；
- 非主键查询必须取得可接受的 EXPLAIN，未知、失败和全表扫描默认拒绝；
- procedure 默认拒绝，仅允许 DBA 审核、显式 trusted 且 fingerprint allowlisted
  的调用；应用不宣称能观测其内部真实扫描量；
- cache、feedback、输入数组、session budget 和事务状态全部有界。

迁移时必须：

1. 将角色名统一为 trim 后的小写，消除规范化碰撞。
2. 移除对 mask 字段 filter/group-by/aggregate/cursor 的依赖。
3. 将 file secret 移到 `server.secrets.allowedRoots`。
4. 为必要 procedure 设置 `mcp.trustedProcedure: true`，审核并加入 CALL
   fingerprint，同时配置数据库用户最小权限。
5. 将预算 `maxScannedRows` 迁移为 `maxEstimatedScannedRows`，并检查新增的响应、
   输入、缓存和事务 timeout 默认值。

完整配置和信任边界分别见
[配置参考](../configuration.md)与[安全模型](../security.md)。
