# v0.1.0

`v0.1.0` 是首个功能基线。发布构建通过 linker 变量注入版本；CLI `version` 与
MCP `Implementation` 共用 `version.String()`。未注入的开发构建显示 `dev`。
本文描述仓库中的实现事实，不代表已经存在对应 Git tag、二进制制品或生产 SLA。

## 包含

- PostgreSQL、MySQL、OceanBase（MySQL 协议）provider，启动 ping 和 schema
  introspection。
- stdio 与 streamable HTTP MCP；HTTP bearer token、TLS/mTLS、安全启动检查。
- describe/read/create/update/delete/execute/aggregate 七个实体工具，以及
  begin/commit/rollback 显式事务工具。
- 参数化关系代数/codegen、RBAC、字段 ACL、subject 行级策略、字段 mask。
- 成本链、查询超时、读取行数 cap、模板 fingerprint、读取反馈校准，以及
  PostgreSQL opt-in 只读 `EXPLAIN ANALYZE` 采样。
- 命名数据源、同源关系展开、offset/keyset、prepared statement cache。
- 角色/租户进程内预算、有界 engine、singleflight、AIMD、熔断、异步审计和
  OpenTelemetry hook。
- 原子 App 热重载、授权 schema resource、三个安全 prompts。
- CLI：`serve`、`init`、`add entity`、`validate`、`explain`、`version`。

## 默认安全行为

- 成本、mask 和 rate limiting 默认开启。
- `delete_record` 默认关闭，其他内置工具（含事务工具）默认开启。
- `requirePKForWrite` 默认开启；update/delete 无有效谓词时始终拒绝。
- HTTP 非 loopback 无 token/mTLS 时拒绝启动。
- 配置引用缺失 table/view/field 或未知 mask 时启动失败。

安全模型的精确定义只在 [security.md](../security.md) 维护。

## 已知限制

- 默认 EXPLAIN Estimate 只对 PostgreSQL 启用。MySQL/OceanBase 有 parser，但
  方言声明估算不够可靠，故不进入同步成本链。
- `EXPLAIN ANALYZE` 采样默认关闭，仅 PostgreSQL 支持；命中采样会额外执行一次
  生成的只读语句。MySQL/OceanBase 不实现 sampler，启用配置时 fail-fast。
- 方言的 statement timeout、scan-row cap、OceanBase resource manager 是能力
  描述，不代表 provider 已执行原生配置。实际启用的是 context timeout 和
  MySQL 协议的 `sql_safe_updates`。
- budget 是单进程内存实现；部分行数/cost 限制在执行完成后判定，不是 DB 硬
  配额。
- 热重载不改变 MCP tools/list 注册集合；工具定义变化需要重启。
- 关系展开仅同数据源、单 join pair、一层；没有通用 join 或跨库 join。
- OAuth、持久 session/event store、默认 metrics exporter/endpoint、审计轮转
  尚未提供。
- `rateLimit.rps` 可解析但未装配。`weak.Pointer`、`unique.Make` 未采用。
- `config` 因 YAML 自定义解码直接依赖 `yaml.v3`，当前不满足“所有核心包仅
  标准库”的绝对表述。

## 发布门禁与测试映射

仓库已实现以下 release gates；“已实现”表示检查存在且会失败关闭，不表示本文
所在 checkout 已在所有环境运行成功：

  govulncheck：`make ci-local`（`make ci` 的别名，不需要 Docker）。

  MCP client e2e：`make ci-full`（需要 Docker）。

  只在 Makefile 定义并执行真实数值比较。

- 授权 schema resource、prompts、custom procedure 调用和 MCP 版本协商：
  `x/mcpserver/features_test.go` 使用稳定的 in-memory transport 验证。

  和 `x/mcpserver/features_test.go`。
