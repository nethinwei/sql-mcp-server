# v0.1.9

`v0.1.9` 交付 Real Business Workload Model：Eval 从单一合成基线升级为
双轨体系，并交付真实负载轨的载体——一组可复现、可扩展、可供社区共同
改进的业务 reference model、确定性数据与自然语言任务
（[设计文档](../design/business-workload-model.md)为范围唯一事实源）。
本版本不新增 IR 能力、Provider 或运行时功能。

## 四模块业务 reference model（`fixtures/v4/`）

行业中立的 `commerce-core`（租户/客户/商品/订单、价格版本、逻辑删除、
users vs customers 相似实体陷阱）、`payment-orchestration`（支付意图与
尝试双 grain、路由规则版本、重复/乱序/未处理回调）、`ledger-settlement`
（复式分录恒归零、余额快照 vs 流水、手续费规则版本、对账四态、支付成功
但未入账），及首个行业扩展 `verticals/live-monetization`（房间/场次
grain、礼物价格与分成规则版本化、重复事件去重）。共 45 个实体，金额
一律整数最小单位（`amount_minor` + `currency_code`，含零小数位 KRW 与
虚拟资产 star）。

确定性生成器（`fixtures/v4/generator`）：所有值是 `(seed, 表, 行号)`
的纯函数，`Scale` 控制事实表规模（维表固定），8 类异常模式逐项注入，
PostgreSQL 与 MySQL/OceanBase 双方言渲染。DDL、seed、expected CSV、
policies 与可直接运行的组合 profile（`fixtures/v4/profiles/default.yaml`）
全部由 `make fixtures-v4` 生成并纳入版本控制；drift 测试锁定磁盘文件与
生成器一致，组合 profile 经真实配置加载器校验。设计文档复杂度清单
12/12 项全覆盖（核对表见 [fixtures/v4/README.md](../../fixtures/v4/README.md)）。

## Eval 双轨化

- **回归轨**（`make eval-pilot`）：v3 任务集与 fixture 冻结为回归基线，
  资产移至 `eval/regression/`（内容零改动）；迁移后验证 32/32 全绿；
- **真实负载轨**（`make eval-workload`）：21 个任务（分布门禁加载时
  机械强制：商业+支付 12、账务 5、直播 4），任务文件不写答案——预期值
  由生成器与 seed 数据同源计算并在加载时注入；
- 评分双通道：机械通道（`expect_tool` + 数字边界匹配，"6" 不误配
  "16"，句末标点是合法边界）与证据通道（预期行在工具返回中的覆盖率 +
  十类失败归因：发现/参数/关系/grain/时间/状态/单位币种/IR 表达/
  Provider 差异/治理策略；结构性失败机械归因，语义失败落任务声明的
  候选归因并标记人工复核）；
- dogfooding 模式：`EVAL_DSN`/`EVAL_CONFIG`/`EVAL_TASKS` 直连外部真实
  或脱敏业务库，问题清单模板见
  [eval/results/dogfooding-template.md](../../eval/results/dogfooding-template.md)。

## 跨 Provider workload 一致性

`internal/conformance.RunWorkload`：45 张表逐表 checksum 聚合
（行数 + 数值列求和 + 可空列非空计数），reference interpreter 与
Provider 经 codegen 编译执行差分比对，接入 `make test-integration`。
验收运行：PostgreSQL、MySQL、OceanBase 全部通过。

## 首轮正式评测（任务集 v4）

deepseek-v4-flash 三轮：**21/21、21/21、21/21**；first-call success
57–62%，repair rate 67–100%；单任务 prompt token 约 18–20K（v3 的
2.6 倍，主因 45 实体 catalog）。归因与 go/no-go 结论
（[完整结论](../../eval/results/2026-07-12-deepseek-v4-flash-workload-v4.md)）：
Semantic Metadata 与 Governed Query Expressiveness 维持 **no-go**，
Catalog Discovery 转**继续观察**（token 成本信号显著但非失败源）。
一例评分产物（数字边界把句号当作小数点延续）已修复并由确定性测试锁定。

## 已知限制

- v4 三轮 100% 通过率说明该任务集对 deepseek-v4-flash 同样接近饱和：
  结论不外推到更弱模型；需要区分度时应使用更弱模型、提高 `Scale` 或
  引入 dogfooding 真实负载；
- 证据通道的行覆盖率是复核信号而非判分：模型侧聚合可合法产生工具
  输出中不存在的数值；
- workload conformance 校验数据一致性（checksum 聚合），查询语义
  等价仍由 v0.1.8 的 conformance corpus 保证；字符串内容与日期值不
  进入 checksum（collation 与日期渲染属已登记偏差）；
- 多跳关联依赖 Agent 多次调用分解（单跳 relationship 展开），调用
  成本可见但未构成失败源。
