# v0.1.1

`v0.1.1` 在 `v0.1.0` 功能基线上做结构整理，不改变对外 MCP 工具契约。

## 变更

- 方言实现（标识符引用、占位符、EXPLAIN 包装、能力声明）迁至
  `x/providers/postgres`、`x/providers/mysql`、`x/providers/oceanbase`；核心
  `dialect` 包仅保留接口与 `Capabilities`。
- 配置 JSON Schema（`core/config/schema.json`）与 MCP 工具 input schema
  （`core/tool/schema/*.json`）通过 `embed` 嵌入，便于编辑与审阅。
- 文档与 CHANGELOG 中先前未发布的变更一并纳入本版本。

## 迁移说明

- 若你的代码直接构造 `dialect.Postgres{}` 等类型，请改为引用对应 provider
  包中的 `Dialect`（例如 `postgres.Dialect{}`）。核心包单元测试使用
  `internal/testdialect` 以避免与 `x/providers` 的 import cycle。
- `dialect.New()` 已移除；运行时应通过 bootstrap 按 driver 打开 provider 并
  从其 `Dialect()` 方法获取方言。
- `config.Schema()` 与工具 schema 变量行为不变，仅实现方式改为 embed。

### 程序化配置迁移

仅使用 YAML 的部署可以跳过本节。直接构造 Go 配置的嵌入方还需要：

- 将 `config.CostConfig.Enabled: true/false` 改为
  `Enabled: config.Bool(true/false)`，读取有效值使用 `EnabledOrDefault()`；
- 多数据源中的精确 SQL baseline 改为 `datasource:SQL`；`fp:v2:` fingerprint
  已包含 datasource，不受影响；
- 热重载可以更新预算限制并保留 session 用量，但事务 `ttl`/`maxOpen` 变化会
  拒绝 reload，必须重启。

## 与 v0.1.0 的关系

功能、默认安全行为、配置字段与 MCP 工具列表与 `v0.1.0` 一致。详见
[`v0.1.0`](v0.1.0.md)。后续 [`v0.1.2`](v0.1.2.md) 在此基础上收紧安全与资源
边界。
