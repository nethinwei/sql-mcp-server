# v0.1.5

`v0.1.5` 交付 Contract + Product Proof：固定数据面的机器契约和真实 HTTP 验证
路径，并把既有安全、发布和 quickstart 能力组合成五分钟可复现的产品证明。它不
新增 Provider、控制面或持久化 revision。

## 机器可读拒绝契约与 decision ID

业务级拒绝现在在 `CallToolResult.structuredContent` 携带稳定契约：`code`、
`reason`、`retryable`、`constraints`、`hints` 和 `decisionId`。decision ID 由
`crypto/rand` 逐调用生成，贯穿 MCP 响应、审计事件（`DecisionID` 字段）与
trace span（`decision.id` 属性），包括预算获取阶段的拒绝。

- 全部 tool schema、错误码表与 Denial 字段名由 golden 快照
  （`core/tool/testdata/contract.json`）在 CI 机器检查；兼容与破坏性变化的
  判定规则见 [tool-contract.md](../tool-contract.md)；
- 修复建议（hints）只能收紧或等价改写请求；
- 预算拒绝从协议层内部错误改为业务级 `BUDGET_EXCEEDED` 结果；
- `UNAUTHORIZED` 的客户端可见 `reason` 统一泛化，不回显实体/字段/角色细节
  （TM-002 防枚举）；详细拒绝原因仅写入审计。机器码仍可区分
  `ENTITY_NOT_FOUND` 与 `UNAUTHORIZED`，该语义为 Agent 自修复保留并在威胁
  模型中登记为已接受的残余；
- 内部/驱动错误不进入该契约，继续以协议层错误返回且不回显细节。

## 真实 streamable HTTP e2e

`make test-e2e` 现在同时覆盖 in-memory 传输和真实 streamable HTTP listener
（bearer token + 可信身份 header），共享同一断言套件以证明关键授权路径在两条
传输上等价，并额外验证认证失败边界、`X-MCP-Subject` 驱动的 tenant 隔离，以及
拒绝响应 `decisionId` 与审计 JSONL 事件的关联。

## 确定性 YAML export

新增 CLI `sql-mcp-server export`：字段顺序固定、默认值由启动同款加载器物化、
secret 占位符（`${ENV}`/`${file:...}`）原样保留。同一配置重复导出字节级一致，
输出可直接通过 `validate`。该格式同时是 revision 设计评审
（[design/revision-snapshot.md](../design/revision-snapshot.md)）选定的持久化
payload 形态；本版本不实现持久化 revision。

## 六场景产品证明与证据索引

- [quickstart](../quickstart.md) 扩展为六场景：正常查询、字段拒绝、跨 tenant
  拒绝、mask 可返回但不可过滤、执行前成本拒绝、Agent 按结构化错误收窄后成功；
  release 链的 quickstart smoke 自动验证全部六场景；
- 新增[客户端接入核对](../clients.md)：四客户端模板核对结论、自动化证据与
  边界声明（未做原生桌面 GUI 人工 smoke，该项不作为保证）；
- 新增[架构、安全边界与证据索引](../evidence.md)：一页边界说明加全部对外声明
  的证据位置。

## 威胁可追溯性

TM-001~TM-008 到回归测试的映射由 `internal/threatcheck/coverage.json` 维护并
在默认单元测试中机器校验：新增 TM ID 未映射、引用测试被删改、条目缺证据均使
CI 失败；剩余缺口显式标注。CI 的 fuzz smoke 仍为每 target 20 秒的有界运行，
不构成长期持续 fuzz。

## 已知限制

- 拒绝契约的 `reason`/`hints` 文案不属于契约，客户端解析必须基于 `code` 与
  `constraints`；
- decision ID 仅覆盖工具调用，schema resource 与 prompt 读取不生成决策记录；
- HTTP e2e 使用进程内 listener 与可信 CIDR 模拟代理，尚无独立反向代理进程
  参与的端到端验证；
- 审计 JSON Lines 字段命名尚未定版（当前为 Go 字段名），计划在下一版本随
  telemetry 工作冻结；
- export 不解析也不脱敏配置中的字面量值；secret 管理仍是配置方责任。
